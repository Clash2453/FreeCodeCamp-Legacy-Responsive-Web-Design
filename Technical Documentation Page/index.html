Intro

                    This is the course work I did for my object oriented programming class, part of my first year of programming in university. The theme of the project is School Management System.
                    The program is inspired by existing platforms such as Moodle and aims to provide a user-friendly graphical interface to the user using Windows Forms technology. The program will provide a set of functionalities to the user depending on his/her position (student, professor, administrator, parent), among which will be: displaying students' grades in the form of a graph and their schedule in the form of an interactive calendar, the ability of administrators and professors to add new events (lectures, seminar exercises and lab exercises), etc.

Features

The program supports 2 class hierarchies: one for the users of the program and one for the different types of events that will be entered into the interactive calendar. All users will inherit the Person base class and its properties, and through the use of virtual methods will perform their own specific functions. The user classes supported by the program are: professor, student, parent, administrator. The second hierarchy of classes includes the different types of calendar events. They all inherit the base class Event and override its functions as needed. All objects inheriting the Person class are stored in a private List (storing objects of type Person) and are encapsulated, the same applies to objects of type Event.
The program supports the functionality of storing and graphically representing student grades in the form using the Oxyplot.Core library. Their gradings will be stored in a separate text file using an ID number unique to the student. Their grade point average, with that of the other students in their class, will also be presented in graph form, allowing their easy comparison. Additionally, the grades will be present in the form of a table containing the name of the subject and the student's grades in that subject.
The program will have a calendar form that will allow the entry of different types of events by professors and administrators with the ability for the type and content of an event to be viewed by students and parents without the ability to make changes by them. Each date will have an OnClick Event that will open a dialog box from where: the type of event and short notes about it can be entered when opened by an administrator or professor, and when opened by a student or parent it will show the organizer of the event, the time, the hour and the short notes left by him. Each day of the calendar will be a separate item created through the User Control and DateTime functionalities, allowing the dynamic loading of future and previous months and their events. All events will be stored in a sheet, and when the window is closed, will be entered into a text document and loaded at the time of form initialization.
